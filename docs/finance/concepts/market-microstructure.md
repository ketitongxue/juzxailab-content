# 市场微观结构

K线是压缩后的"结果"数据，订单簿（盘口）才是微秒级"意图"数据。本页聚焦普通人如何用盘口信息**防御**与**省滑点**，而非参与高频进攻。K线看历史，盘口看意图。

## 1. 订单簿 Order Book

Level 2 数据展示买卖各档挂单的深度与流动性。当前价 2.0 元，但买二 1.8 元挂着 2000 斤大单——这是**支撑墙（厚尾）**，意味着短期下跌空间有限，AI 可尝试低吸。

- **K线**：告诉你刚才一斤白菜成交 2 块（历史）。
- **订单簿**：告诉你摊主挂多少、顾客喊多少、谁在等谁（意图）。

数据质量与延迟问题见 [market-data-quality](/finance/concepts/market-data-quality)。

## 2. 冰山订单（Iceberg Order）

大机构要卖 100 万股，若直接挂盘口会吓跑散户、砸低自身成交价。**拆单算法**只显示 10 手，被吃掉立刻从库存补充，如打地鼠永远打不完——水面只露一角，水下藏巨体。

### AI 识别方法
1. 监控某价位（如 10.00 元）；
2. 计算该价位累计成交量；
3. 若累计成交 ≫ 盘口显示挂单量（显示100股却成交1万股且价未变）；
4. 报警：发现冰山卖单。

> **应对原则**：识别冰山**不是为套利，而是为避险**。一旦发现上方有冰山卖单，K线形态再好也要等融化后再进场。

## 3. 幌骗（Spoofing）

非法但高频世界屡禁不止的操纵：在买二/买三瞬间挂巨额假单制造"托单很强"的假象，诱多后在你成交的那一微秒撤单，支撑消失价格自由落体。**本质是"不想买装作想买"**，与冰山（想卖装作不想卖）相反。

### AI 识别
核心指标：**订单撤销率**。正常交易者撤单率约 10%，幌骗策略撤单率高达 95% 以上。若盘口常现大单却每次临成交前消失，即机器在钓鱼。**AI 指令**：忽略这些虚假深度，不计入支撑位。

## 4. 做市商与 OI 多空失衡指标

做市商是市场润滑剂，靠买卖价差盈利，但面临逆向选择风险（被知情交易者砸盘）。

### 买卖深度不平衡 Order Imbalance (OI)

```
OI = (买一挂单量 - 卖一挂单量) / (买一挂单量 + 卖一挂单量)
```

OI ∈ [-1, 1]：

- **≈ 0**：买卖势均力敌，震荡市，适合做 [strategy-trio-trend-grid-momentum](/finance/concepts/strategy-trio-trend-grid-momentum) 中的网格。
- **→ -1**：抛压极重，盘口瞬间失衡。

当 OI 突然从 0 跳到 -0.8/-0.9，说明巨量抛压涌现，做市商会撤单逃跑导致买盘真空、价格瀑布式下跌。**策略：立刻停止抄底/网格，不接飞刀。**

> ⚠️ **延迟警告**：普通软件 Level 2 数据有数十毫秒至 3 秒延迟（快照模式）。**切勿用 OI 写毫秒级自动打单策略**，机构早已跑完。OI 仅用于分钟级趋势判断：是该买还是该等。

## 5. VWAP 算法的反向工程

机构买入 1 亿股而不惊动市场，会用 VWAP（成交量加权平均价）算法把单子拆碎隐藏在自然流动性里。

### AI 识别主力吸筹
若某股票呈现：
1. 价格波动极小（**织布机走势**）；
2. 成交量放大时价格微微向上跳；
3. 成交量缩小时价格横住；

这往往是 VWAP 吸筹特征——全天大资金托底，算法未跑完（通常持续整天）股价就难深跌。可作为持安心持有的定心丸，与主力共舞。

## 6. 普通人的定位：防御 + 省滑点

普通人**不要试图用家用电脑与把服务器架在交易所机房的机构抢速度**——高频（毫秒级）是硬件和光缆的战争，中低频（分钟/日线级）才是逻辑与认知的主场。

### 防御（少亏即赚）
- 看到冰山压顶 → 不买；
- 看到毒性订单流（OI 失衡）→ 撤单；
- 看到幌骗撤单 → 忽略虚假支撑。

### 省滑点（Tick 数据）
普通人唯一能直接获利之处。**不要总挂市价单**，利用 QMT 的 Tick 数据看支撑墙位置：
- 要买入时，挂在支撑墙前一位（插队）。
- 长期下来每笔省 0.1% 滑点，一年累积即显著超额收益。

> **盘口数据是宏观决策的验证码，而不是发令枪。**
