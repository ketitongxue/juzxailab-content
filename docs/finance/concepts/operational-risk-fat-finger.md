# 操作风险与乌龙指

2013 年 8 月 16 日，光大证券的 ETF 套利系统失控，短短几分钟发出 **234 亿元**买单，实际成交 72.7 亿元。这不是救市，是一场代码灾难——量化系统脆弱性的教科书案例。

## 光大乌龙指事件（2013）

11:05 分，中国石油、中国石化、工商银行等平时日波动不到 1% 的巨无霸瞬间直线拉升，11:06 几十只权重股集体涨停。上证指数**两分钟内暴涨 5.6%**（超 100 点）。全中国股民以为国家队救市，但在光大证券策略投资部交易室，只有死一样的寂静。

## 策略本无罪：ETF 申赎套利

光大做的是经典低风险量化策略——ETF 申赎套利：

1. ETF 有两个价格：二级市场价格 和 IOPV（净值）。
2. 当二级市场价格 > IOPV（溢价），说明 ETF 贵了。
3. 买入一篮子股票 → 申购 ETF → 二级市场卖出 → 赚差价。

这是一个完美的闭环：风险低、拼速度、拼自动化。

## 致命 Bug：死循环

订单生成模块有一个**重下（Retry）**功能。当系统检测到部分股票"没有成交"（其实是回报延迟或停牌），程序员写的逻辑判断错误被触发：系统认为"这笔单子完全没成交"，于是**把整个篮子再买一遍**。

检测-重发间隔极短且无限制。致命的 2 秒钟内，计算机生成了 **26082 笔委托**。就像你买火车票狂点 100 下，系统不是提示"请勿重复提交"，而是真的从你银行卡扣了 100 次钱。

## 三重风控缺位

| 风控环节 | 应有状态 | 光大实际 |
|----------|----------|----------|
| 资金前端控制 | 下单前检查可用资金 > 订单金额 | 申请了交易所直连模式，**不再校验资金余额** |
| 订单量限制 | 单笔/单日上限（如单笔不超 100 万） | 参数为空或设为**无限大** |
| Kill Switch | 物理一键切断按钮 | **找不到停止按钮**，慌乱中每秒流出去上亿 |

> 在追求速度的道路上，如果抛弃了刹车，你跑得越快，死得越惨。

## 从技术事故到法律灾难

11:07 系统停下后，光大看着满手多出的 72 亿股票（收盘后要交割，否则破产），做出决定：**对冲**。

当天下午在没有向公众披露乌龙指的情况下：
- 期货市场疯狂**卖空** IF 股指期货（总计 43 亿元）。
- ETF 市场卖出 ETF。

在监管和散户眼里，这是**内幕交易**——你知道上午暴涨是假的，知道下午大概率会跌，却利用信息优势反手做空。

**结局**：没收违法所得 + 5 倍罚款，总计罚没 **5.2 亿元**；相关高管终身市场禁入。

## AI 时代的新乌龙指

传统乌龙指是逻辑循环，AI 的乌龙指是**幻觉循环**：

1. Agent 看到盘口大卖单（冰山订单），误判为主力出货、马上跌停。
2. 不计成本市价抛售。
3. 它的抛售把股价砸下去了。
4. Agent 反馈："看！股价真的跌了！我的判断对！必须卖得更快！"
5. **死循环**：Agent 自己吓自己，把股价砸到跌停。

这种自我实现的乌龙指符合自己的闭环逻辑（虽然是错误的），在 AI 自动交易中极难防范。

## 防弹衣代码四件套

不管策略多赚钱，必须加上以下代码：

**1. 硬性资金限额**
```python
MAX_DAILY_BUY = 1000000  # 每天最多买100万
if current_buy_amount > MAX_DAILY_BUY:
    raise Exception("触发熔断：今日买入超限！")
```

**2. 单笔订单限制**
```python
MAX_ORDER_VALUE = 50000  # 单笔不能超过5万
if price * volume > MAX_ORDER_VALUE:
    print("警告：单笔订单过大，已拦截")
    return
```

**3. 死循环熔断**
```python
retry_count = 0
while not order_filled:
    retry_count += 1
    if retry_count > 5:
        print("错误：重试次数过多，停止下单！")
        break
```

**4. 物理开关**：手机里有一个脚本，一键杀掉服务器上所有 Python 进程。看到异常，不要改代码，先按这个按钮。

参见 [ai-quant-agent-workflow](/finance/concepts/ai-quant-agent-workflow) 的双重安全锁设计。

## 核心教训

- **敬畏每一行代码**：Code is Money，Bug 可能导致破产。
- **模拟盘不等于实盘**：模拟盘没有拥堵、延迟、部分成交。永远不要拿实盘资金帮代码 Debug。
- **保留人类监控**：哪怕全自动策略也要设心跳报警，程序 1 分钟交易超 10 次必须短信轰炸手机。
