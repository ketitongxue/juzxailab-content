# 智能体会话持久化

智能体会话持久化把对话状态保存为可检查的历史，使会话能够恢复、分叉、导出和安全回放。

## 树状消息历史

一种实用结构是 JSONL：文件头记录版本，后续每条记录包含条目 ID、父条目 ID 和消息。父子关系让历史天然形成树，普通对话只是其中一条根到叶路径。

它支持两种不同操作：

- **追加分支**：从指定叶节点继续，同时保留原有树。
- **复制会话**：把一条根到叶路径复制为独立会话。

如果每次追加都把历史重新展平并生成新 ID，就会破坏原有分支身份。

## 存储与恢复

- 写入临时文件后原子重命名，避免读到半写入状态。
- 使用版本化读取器集中迁移旧格式。
- 复用[智能体运行时契约](/wiki/concepts/agent-runtime-contracts)中的消息模型。
- 恢复时检查缺失父节点和循环引用，损坏数据应显式报错。
- 让工具结果、多模态内容和[上下文压缩](/wiki/concepts/agent-context-compaction)检查点保持原始语义。

## 导出与安全

机器交换需要无损 JSONL；人工检查可以生成自包含 HTML 回放。HTML 应内联样式、避免远程脚本，并转义所有模型和工具文本，防止历史内容成为注入入口。

持久化后的会话不只是“记住聊天”，也为调试、评估、审计和跨进程恢复提供了可复现证据。
